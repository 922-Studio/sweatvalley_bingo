  <name>Task 2: Configure Cloudflare Tunnel and DNS</name>
  <files>N/A (external setup)</files>
  <action>
Server and client are configured for bingo.922-studio.com. Now the network routing needs to be set up.

The user needs to do two things:

**A. Cloudflare Tunnel ingress rule (on the server):**
1. Find cloudflared config: `systemctl cat cloudflared` or `ps aux | grep cloudflared` to find the `--config` path
2. Edit the config.yml and add before the catch-all rule:
   ```yaml
   - hostname: bingo.922-studio.com
     service: http://localhost:3923
   ```
3. Validate: `cloudflared tunnel ingress validate`
4. Restart: `sudo systemctl restart cloudflared`

**B. Cloudflare DNS CNAME record:**
1. Go to Cloudflare Dashboard -> 922-studio.com -> DNS
2. Add record: Type=CNAME, Name=bingo, Target=TUNNEL-UUID.cfargotunnel.com
3. Ensure Proxy status is "Proxied" (orange cloud), NOT "DNS only"
4. The tunnel UUID can be found in the cloudflared config or via `cloudflared tunnel list`

**Verify:**
- `curl -f https://bingo.922-studio.com/health` should return `{"status":"ok"}`
- Open https://bingo.922-studio.com in browser -- app should load and WebSocket should connect
  </action>

Create `docs/cloudflare-tunnel-guide.md` documenting the pattern for adding new services under 922-studio.com subdomains via Cloudflare Tunnel. This is the DOCS-01 requirement for HomeStructure docs.

The guide should cover:

1. **Overview** -- What Cloudflare Tunnel does (exposes local services without port forwarding, handles SSL)

2. **Prerequisites** -- cloudflared installed and running, tunnel already created, DNS zone for 922-studio.com in Cloudflare

3. **Steps to add a new subdomain** (e.g., `newservice.922-studio.com`):
   - Step 1: Find the cloudflared config file location (`systemctl cat cloudflared`)
   - Step 2: Add an ingress rule BEFORE the catch-all:
     ```yaml
     - hostname: newservice.922-studio.com
       service: http://localhost:PORT
     ```
   - Step 3: Validate config: `cloudflared tunnel ingress validate`
   - Step 4: Add CNAME in Cloudflare DNS dashboard:
     - Type: CNAME
     - Name: newservice
     - Target: TUNNEL-UUID.cfargotunnel.com
     - Proxy: Proxied (orange cloud) -- MUST be proxied, not DNS-only
   - Step 5: Restart cloudflared: `sudo systemctl restart cloudflared`
   - Step 6: Verify: `curl -f https://newservice.922-studio.com`

4. **Current services** -- Table of existing subdomain mappings:
   | Subdomain | Local Port | Service |
   |-----------|-----------|---------|
   | 922-studio.com | 8010 | Main site |
   | gregor.922-studio.com | 3922 | Gregor's site |
   | bingo.922-studio.com | 3923 | Schweisstal Bingo |

5. **Troubleshooting** -- Common issues:
   - CNAME not proxied (grey cloud) -> tunnel won't receive traffic
   - Forgot catch-all rule at end -> cloudflared won't start
   - Service not running on expected port -> 502 Bad Gateway
   - Config file in wrong location -> changes don't take effect after restart

Keep it concise and practical. This is a reference guide, not a tutorial.
