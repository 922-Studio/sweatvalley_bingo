Three changes in server/server.js:

1. **createGame function (line ~88):** Add `maxRounds` parameter with default 1. Validate with Math.max:
```javascript
function createGame(gameId, hostId, hostName, gridSize = '4x4', maxRounds = 1) {
  const game = {
    // ...existing fields unchanged...
    maxRounds: Math.max(1, parseInt(maxRounds, 10) || 1)
  };
}
```

2. **create-game handler (line ~120-123):** Read maxRounds from data and pass to createGame:
```javascript
socket.on('create-game', (data) => {
  const gameId = Math.random().toString(36).substr(2, 9);
  const gridSize = data.gridSize || '4x4';
  const maxRounds = parseInt(data.maxRounds, 10) || 1;
  const game = createGame(gameId, socket.id, data.hostName, gridSize, maxRounds);
  // ...rest unchanged...
});
```

3. **game-started emit (line ~202):** Include maxRounds in the event payload:
```javascript
io.to(gameId).emit('game-started', { playerGrids, gridSize, maxRounds: game.maxRounds });
```

Do NOT change any other server logic. The end-round handler already correctly uses `game.maxRounds` (line 307).
